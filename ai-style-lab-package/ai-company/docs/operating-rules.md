# 全社共通ルール（ai-style-lab / AIカンパニー）

## 0. 混ぜない（永久ルール）
既存プロジェクトの下に置いても、**AIカンパニーの内容を本体と混ぜない**。
codex は `ai-style-lab` の外のファイルを編集しない。

## 1. ネガティブ表現を使わない
弱点／向いていない／劣る／失敗 などは使わない。持ち味・伸びしろ・前向きな言い回しに置き換える。

## 2. 役割で動く
はやと → AI部長（一次判断）→ つばさ（整理・振り分け・報告）→ 各担当（codex実行）。
実行しっぱなしにせず、必ず結果を報告する。

## 3. 確認のルール
外部公開・お金・アカウント・不可逆な操作・方針変更は、実行前に **hayato の最終判断**を取る。
迷ったら一問一答で確認してから動く。

## 4. スキル優先
決まった作業は `skills/` の該当スキル通りに実行する。
スキルが無い作業をやったら、**終わったあとにスキル化を提案**して積み上げる。

## 5. 記録
実行したら `docs/worklog.md` に1行追記。判断は `ai-bucho/decision-log.md` に残す。

## 6. 出力品質（暴走・反復対策）
- ツール実行の前後に埋め草テキストを書かない
- 同じ言葉・同じ行を繰り返さない。返答は短く要点だけ
- 異常時は hayato の「ストップ」で即停止

## 7. PR / コミット
PR は hayato が明示的に依頼した時だけ作る。コミットメッセージは何をしたか簡潔に。
