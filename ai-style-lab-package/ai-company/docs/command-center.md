# 司令塔 — codex を動かす仕組み（ai-style-lab）

はやと → つばさ（整理・振り分け）→ **codex が実行**。
codex は Cursor 内で動き、このリポの `ai-company/` を読んで指示に沿う。

## 連携レベル
| レベル | 仕組み | 状態 |
|--------|--------|------|
| L1 指示書方式 | つばさが作業指示書を作る → codexに貼って実行 | ✅ 今すぐ |
| **L2 掲示板方式** | 指示を `ai-company/` のファイルに書く → codexがルート `AGENTS.md` 経由で読む | ✅ **この構成で稼働** |
| L3 直結 | API/MCPで自動起動 | 🔵 将来 |

## codex が読む順（L2の要）
1. ルート `AGENTS.md`（codexが自動で読む入口）
2. `ai-company/ai-bucho/hayato-directives.md`（はやとの方針）
3. `ai-company/skills/` の該当スキル
4. `ai-company/company/ai-style-lab.md`（社の役割）

## 標準フロー
1. **受ける**：用件を1文で復唱
2. **判断**：AI部長が一次判断（governance参照）
3. **スキル確認**：`skills/` に該当があればその手順で実行
4. **実行**：codex が作業（ai-style-lab 内のみ）
5. **記録・報告**：worklog に追記 → はやとへ「件名・結果・変更点」を報告

## 作業指示書スキーマ
```
【プロジェクト】ai-style-lab（AIスタイル診断）
【ゴール】1文で「何が完成すれば成功か」
【前提・入力】参照ファイル・素材
【使うスキル】skills/◯◯（あれば）
【要件】箇条書き
【完了条件(DoD)】これが満たされたら完了
【報告】結果・変更ファイル・詰まった点
```
