# 顔立ち写真観察データ設計

## 1. 目的

顔写真から Pop / Clear / Bloom / Mode の候補を出す前に、写真品質と観察特徴を再現可能なデータとして分離する。

判定は画像AIの自由文に丸投げせず、次の順序で行う。

1. 写真品質の確認
2. 顔ランドマークから観察値を算出
3. 算出できない項目は「不明」として保留
4. 子ども寄り / 大人寄りの7項目を多数決
5. 曲線寄り / 直線寄りの8項目を多数決
6. 2軸の組み合わせから4候補を生成
7. 判定材料の充足度をConfidenceとして別途表示

> 製品表示では「顔タイプ診断」という呼称を使わず、「顔立ち傾向チェック」とする。

## 2. MVPで使う写真

| 写真 | MVPでの扱い | 用途 |
|---|---|---|
| 正面・自然な表情 | 必須 | 幾何比率と15項目の中心計測 |
| 笑顔 | 推奨 | 表情変化による印象差の確認。多数決の直接票にはしない |
| 左右のななめ写真 | 将来拡張 | 正面だけでは取りにくい立体感を補助 |

正面写真が利用できない場合は候補率を出さず、撮り直しを案内する。

## 3. 写真品質データ

### 3.1 スキーマ

```ts
type FacePhotoQuality = {
  status: 'usable' | 'limited' | 'retake';
  faceDetected: boolean;
  singleFace: boolean;
  frontality: number;       // 0..1
  visibility: number;       // 0..1
  sharpness: number;        // 0..1
  brightness: number;       // 0..1
  lightingUniformity: number; // 0..1
  occlusion: number;        // 0..1, 高いほど遮蔽が多い
  warnings: FacePhotoWarning[];
};
```

### 3.2 初期しきい値

| 項目 | usable | limited | retake |
|---|---:|---:|---:|
| 顔検出 | 1人 | 1人 | 0人または複数 |
| 正面度 | 0.85以上 | 0.70以上 | 0.70未満 |
| 顔の可視率 | 0.90以上 | 0.80以上 | 0.80未満 |
| ピント | 0.70以上 | 0.50以上 | 0.50未満 |
| 明るさ | 0.30〜0.85 | 閾値の前後10% | それ以外 |
| 遮蔽率 | 0.10以下 | 0.25以下 | 0.25超 |

しきい値は実機テスト前の仮値とし、ラベル付き検証写真で調整する。一つでも `retake` 条件に該当したら、判定工程へ渡さない。

## 4. 観察値の共通形式

```ts
type FaceObservationValue = 'child' | 'adult' | 'curve' | 'straight' | 'unknown';

type FaceObservation = {
  cue: FaceChildAdultCue | FaceCurveStraightCue;
  axis: 'childAdult' | 'curveStraight';
  value: FaceObservationValue;
  measurement?: number;
  measurementUnit?: 'ratio' | 'degree' | 'normalized';
  source: 'landmark' | 'photoModel' | 'selfReport';
  reliability: 'high' | 'medium' | 'low';
  photoType: RequiredPhotoType;
  reasonCode: string;
};
```

`measurement` は後から検証できる生の観測値、`value` は多数決に渡す離散値とする。しきい値付近は強制的に `unknown` とし、僅差を一票に変換しない。

## 5. 子ども寄り / 大人寄りの7項目

| cue | 観察特徴 | child | adult | 自動観測 | 信頼度 | MVPでの扱い |
|---|---|---|---|---|---|---|
| `faceShape` | 顔の縦横比 | 横幅寄り・短め | 縦長寄り | ランドマーク | 高 | 有効票 |
| `chinLength` | あご長 / 顔長 | 短め | 長め | ランドマーク | 高 | 有効票 |
| `eyeDistance` | 両目間 / 片目幅 | 間隔が広め | 間隔が狭め | ランドマーク | 高 | 有効票 |
| `noseHeight` | 鼻の高さ・立体感 | 控えめ | 立体感 | 単眼疑似3D | 低 | 原則 `unknown` |
| `dimension` | 顔全体の立体感 | 平面寄り | 立体寄り | 単眼疑似3D | 低 | 原則 `unknown` |
| `eyeSize` | 目の面積 / 顔面積 | 大きめ | 控えめ | ランドマーク | 高 | 有効票 |
| `nostrilWidth` | 小鼻幅 / 目の横幅 | 目幅1つ分以下寄り | 目幅1つ分超寄り | ランドマーク | 高 | 有効票 |

MVPでは5項目を自動票とし、`noseHeight` と `dimension` は検証が完了するまで多数決から外す。

## 6. 曲線寄り / 直線寄りの8項目

| cue | 観察特徴 | curve | straight | 自動観測 | 信頼度 | MVPでの扱い |
|---|---|---|---|---|---|---|
| `overallBone` | 輪郭の曲率と角 | 滑らかな丸み | 角が明確 | ランドマーク | 中 | 有効票 |
| `cheekVolume` | 頬部の曲率・膨らみ | 丸みが見える | 控えめ | ランドマーク | 中 | 境界は `unknown` |
| `eyeShape` | 目の縦横比と上下まぶた曲率 | 丸み | 横長・直線寄り | ランドマーク | 高 | 有効票 |
| `eyelid` | まぶたの線の形 | 曲線的 | 直線的 | ランドマーク | 中 | 検出安定時のみ有効 |
| `eyeCorner` | 内眼角→外眼角の角度 | たれ目寄り | つり目寄り | ランドマーク | 中 | 有効票 |
| `eyebrow` | 眉の曲率 | アーチ・曲線 | 直線寄り | ランドマーク | 中 | 前髪などの遮蔽時は `unknown` |
| `noseTip` | 鼻先輪郭の曲率 | 丸み | 直線・シャープ | 画像モデル併用 | 低 | 原則 `unknown` |
| `lipThickness` | 唇厚 / 顔長 | 厚みあり | 薄め | ランドマーク | 高 | 有効票 |

MVPでは6項目までを自動票とし、`noseTip` は検証できるまで多数決から外す。`eyelid` と `eyebrow` は検出状態が安定した場合だけ票として採用する。

## 7. 多数決ルール

### 7.1 軸判定

- 2軸を独立に集計する。
- 信頼度による重み付けはしない。有効な項目はすべて1票とする。
- `unknown` は母数に入れない。
- 有効票が各3票未満の軸は `insufficient`。
- 同数の軸は `mixed`。
- 1票差は候補の境界とし、2候補を表示する。

### 7.2 4候補への変換

| 子ども/大人 | 曲線/直線 | 候補 |
|---|---|---|
| child | curve | Pop |
| child | straight | Clear |
| adult | curve | Bloom |
| adult | straight | Mode |

片方の軸が `mixed` または1票差の場合は2候補、両軸とも境界の場合は4候補を保持する。候補率は正解率と呼ばず、各2軸の票比率から作る「現在の写真との一致度」とする。

## 8. Confidenceの設計

Confidenceはタイプ候補の割合と分け、判定材料の状態を表す。

```text
qualityScore     = 必須写真の品質平均
coverageScore    = 有効票数 / 15
consistencyScore = 同条件で再撮影した場合の一致率。単写真MVPは0.60固定
marginScore      = 2軸の各票差を0..1に正規化した平均

confidence = round(100 * (
  qualityScore * 0.35 +
  coverageScore * 0.30 +
  consistencyScore * 0.20 +
  marginScore * 0.15
))
```

MVPの自動観測は未検証のため、Confidence上限を70とする。ラベル付き検証が完了するまでは、80以上の表示をしない。

| Confidence | 表示 |
|---:|---|
| 0〜29 | 候補率を表示せず、撮り直しまたは追加情報を案内 |
| 30〜49 | 参考候補と追加情報を表示 |
| 50〜69 | 1〜2候補と根拠を表示 |
| 70〜79 | 検証データ導入後に利用 |
| 80〜100 | 専門家検証後まで使用しない |

## 9. 根拠表示

根拠は「観察事実」「解釈」「候補」を分けて保存する。

```ts
type FaceEvidence = {
  observations: Array<{
    cue: FaceChildAdultCue | FaceCurveStraightCue;
    measurement?: number;
    value: FaceObservationValue;
    reasonCode: string;
  }>;
  axisVotes: {
    childAdult: { child: number; adult: number; unknown: number };
    curveStraight: { curve: number; straight: number; unknown: number };
  };
  candidates: Array<{
    key: FaceType;
    agreement: number;
  }>;
  confidence: number;
};
```

表示文は `reasonCode` から定型文を組み立て、画像AIに自由に断定文を書かせない。

> 正面写真で、顔の縦横比と目元の丸みが共通して見えやすいため、現在は Pop 寄りの傾向が見えています。

## 10. 判定を保留する条件

- 正面写真の品質が `retake`
- どちらかの軸が有効3票未満
- Confidenceが30未満
- 顔の大幅な遮蔽、強い俯き・あおり、強い魚眼・広角歪みがある
- 自動観測と本人の回答が各2軸で反対し、追加確認ができない
- 複数写真を使う将来フェーズで、同一cueの結果が過半数で一致しない

案内は「判定できません」ではなく、次の確認行動を1つに絞る。

- 正面度が低い → 「カメラと目線を同じ高さにして、もう1枚撮影する」
- 光が不均一 → 「明るい窓に正面を向けて、もう1枚撮影する」
- 有効票不足 → 「不明の項目に関する補助質問に答える」
- 境界候補 → 「両方の持ち味を表示し、まず試したい方を選ぶ」

## 11. 実装への分割

1. `src/lib/vision/facePhotoQuality.ts`
   - 写真品質の正規化と `usable / limited / retake`
2. `src/lib/vision/faceLandmarks.ts`
   - MediaPipeの出力をプロジェクト内の中間形式へ変換
3. `src/lib/vision/faceObservations.ts`
   - 生の比率・角度から `FaceObservation[]` を作成
4. `src/lib/diagnosis/faceMajority.ts`
   - 既存の多数決に観察値を入力する。多数決自体は維持
5. `src/lib/diagnosis/faceConfidence.ts`
   - 写真品質、有効票数、一貫性、票差からConfidenceを計算

## 12. 検証条件

- 同じ写真は毎回同じ観察値と候補を返す
- 境界値の前後で結果が急に断定的にならない
- `unknown` が多い場合はConfidenceが下がる
- 左右反転で本質的な判定が変わらない
- 背景の変更で顔の幾何指標が変わらない
- 画像をサーバーに保存しないモードで完結できる
- Pop / Clear / Bloom / Modeの表示は常に傾向表現になる
- 製品UI・型名・コメントに制限対象の呼称を出さない

## 13. 未決定事項

次の値は、ラベル付き写真の検証なしに固定しない。

- 各観測値を child / adult / curve / straight に変換する数値しきい値
- 写真品質スコアの本番しきい値
- Confidenceの本番係数
- 年齢、性別、人種、カメラ特性による計測偏りの補正方法
- 検証に必要な写真枚数とラベルの正本

しきい値を決める前に、スタイリスト用ラベリング画面の項目と同じcue名で検証データを蓄積する。
