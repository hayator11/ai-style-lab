# スタイリスト・ラベリング画面 最小仕様（データ収集装置の本体 / 2026-07-07）

> 目的：プロが画像1枚に骨格/顔/カラーの**正解ラベル**を素早く付け、`Label(source=stylist, isGroundTruth=true)` を1件生む。
> ＝教師データ生成装置。前回指摘「データが集まらない」への実体的な答え。
> 依存：`ベースデータ_判定基準.md`（入力項目の元）／`datamodel拡張_ラベル収集.md`（保存先）／`タイプ体系案.md §7/§8`（独自名key）。
> 置き場：`ai-style-lab` の `src/app/admin`（既存の管理画面土台を拡張）。MVPはDBなし・ローカル/仮ストアで可。

---

## 1. 画面構成（1画面・3ペイン）
```
┌───────────────┬───────────────────────────┐
│ ①画像ペイン       │ ②ラベリングペイン（骨格/顔/カラーのタブ）  │
│ 顔・全身・カラー用   │  特徴チェック → タイプ確定 → 確信度        │
│ を並べて表示       │                             │
│ 撮影条件バッジ     ├───────────────────────────┤
│                │ ③保存バー：この画像の3軸ラベルを保存→次へ    │
└───────────────┴───────────────────────────┘
```
- 左に未ラベル画像キュー（進捗：残り◯件／ラベル済み◯件）。
- 画像に撮影条件バッジ（自然光/加工なし/全身写り 等＝`shootingConditions`）。条件NGは「判定不可」で飛ばせる。

## 2. 操作フロー
1. キューから未ラベル画像を開く（顔/全身/カラー用がセットで表示）。
2. **骨格タブ**：特徴を数クリック → 候補タイプが自動サジェスト → プロが確定 → 確信度スライダー。
3. **顔タブ**：2軸（世代感・曲線直線）を置く → 象限からタイプ自動サジェスト → 確定 → 確信度。
4. **カラータブ**：4軸を選ぶ → シーズン自動サジェスト → 確定 → 確信度（照明NGなら「判定不可」）。
5. **保存**：3軸ぶんの `Label` を生成（source=stylist, isGroundTruth=true, observedFeatures付き）、`Images.hasGroundTruth=true`。→ 次の画像へ。

## 3. 各軸の入力UI（ベースデータ→クイック入力）
### 骨格（→ Line/Curve/Frame）
- 重心：上／下／均等　｜鎖骨：目立たない／くっきり／太く目立つ　｜上半身の厚み：厚い／薄い／標準
- 関節・骨：目立たない／小さい／大きい　｜肌質：ハリ／やわ／乾いた
- → 選択に応じて Line/Curve/Frame を候補%で提示。プロが最終確定（1つ or 複数候補）。

### 顔（→ Pop/Clear/Bloom/Mode）
- **世代感**スライダー：子ども顔 ↔ 大人顔　｜**曲線/直線**スライダー：曲線 ↔ 直線
- （任意の詳細トグル：あご長さ・目の形・頬肉感・鼻・唇＝根拠の記録用）
- → 象限から Pop/Clear/Bloom/Mode を自動サジェスト → 確定。

### カラー（→ Sunrise/Mist/Amber/Midnight）
- ベース：イエベ／ブルベ　｜明度：高／低　｜彩度：高／低　｜清濁：清／濁
- 補助：瞳明度・髪明度・白目黒目コントラスト・ゴールド/シルバー
- → 4軸からシーズン自動サジェスト → 確定 or「判定不可（写真条件）」。

## 4. 出力（保存されるデータ）
- 軸ごとに `Label`：`{ imageId, axis, typeKey(独自名), source:'stylist', labelerId, confidence, isGroundTruth:true, observedFeatures:[チェックした特徴キー], note }`
- 迷い（候補が近い）時は複数Labelを許容（confidenceで差をつける）→ 「相性/近接の実証」データにもなる。

## 5. 効率化（プロが数をこなせる工夫）
- キーボード操作（数字キーで選択・Enterで保存・→で次）。
- 直前画像の撮影条件・ラベラーIDを引き継ぎ。
- 1画像あたり目標30〜60秒。1セッションで数十件を目安。

## 6. 進捗・品質管理（後の検証に効く）
- ダッシュボード：軸別のラベル済み件数／ラベラー別件数／「判定不可」率。
- 同一画像を複数プロがラベル→**プロ間一致率**を出せる（正解データの信頼性指標）。
- 将来、AI推定が入ったら §datamodel `AgreementLog` で AI vs プロ を自動照合。

## 7. MVP範囲（作る/作らない）
- 作る：画像表示・3軸クイック入力・自動サジェスト・保存・キュー・進捗。
- 作らない（後回し）：本番認証（当面はプロ用の簡易アクセス）、Supabase接続（ローカル/仮ストアでLabelを積む）、AI推定連携。
